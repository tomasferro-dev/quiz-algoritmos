REGLAS DE FIRESTORE — sesiones por curso + historial
====================================================

Pegá esto en Firebase → Firestore Database → pestaña "Reglas" → Publicar.
Reemplaza CUALQUIER regla anterior.

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /control/{course} {
      allow read: if true;
      allow write: if true;

      match /sessions/{session} {
        allow read, write: if true;
      }

      match /runs/{session}/{quiz}/{score} {
        allow read: if true;
        allow create: if request.resource.data.score is number
                      && request.resource.data.name is string
                      && request.resource.data.name.size() <= 30;
        allow update: if false;
        allow delete: if true;
      }
    }
  }
}

Qué permite:
- Leer estado de sesión, lista de sesiones y rankings (todos).
- Iniciar/terminar/eliminar sesiones (profe).
- Guardar puntajes nuevos; eliminar puntajes (al borrar una sesión o podar las viejas).
- Nadie puede EDITAR un puntaje ya guardado.

NOTA DE SEGURIDAD (igual que antes):
El login de profe es del lado del navegador (no usa Firebase Auth), así que
las operaciones de profe (iniciar/terminar/eliminar sesiones) están abiertas a
nivel reglas. Para un uso de aula es aceptable. Si querés blindarlo, se migra el
login de profe a Firebase Authentication y se restringe el write a ese usuario.
